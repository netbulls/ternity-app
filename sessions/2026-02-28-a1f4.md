### Login redirect loop — root cause and fix

- Investigated user's report: app stuck on "redirecting to login".
- Found local port collision behavior between IPv4/IPv6 on `:3001`:
  - `http://localhost:3001` resolved to IPv6 (`[::1]`) and hit a ProovX dev server page.
  - `http://127.0.0.1:3001` correctly reached Ternity Logto (OIDC endpoint).
- Because both web and API envs used `localhost` for Logto endpoint, auth flow was pointing at the wrong service on IPv6 and looping.
- Fixed by switching endpoints to IPv4 explicitly:
  - `apps/web/.env.local`: `VITE_LOGTO_ENDPOINT=http://127.0.0.1:3001`
  - `apps/api/.env.local`: `LOGTO_ENDPOINT=http://127.0.0.1:3001`
- Restarted local web and API dev servers.
- Verified health:
  - Web `:5173` -> 200
  - API `:3010/health` -> 200
  - Logto token POST on `127.0.0.1:3001/oidc/token` -> 200
