# Session 2026-03-07-5f9b

## Topic: Reports System Planning & Analysis

**What the user wants:**

- Plan the implementation of a reports system for sending reports to customers
- Query capabilities: filter by project, team, time range, and other entry parameters
- Saved report templates that can be reapplied, modified, and saved as new favorites
- Company-branded PDF templates showing:
  - Monthly summary
  - All entries by each user during selected period
  - Pie chart showing % composition of each user's contribution
- Question: Should this live in a standalone "Reports" module or extend the existing Entries capabilities?
- Wants 5 proposals for report generation feature approach
- Wants 5 proposals for PDF template generation approach
- Wants analysis of the most appropriate approach overall

## Decisions Made

1. **Reports as a dedicated module** — separate from Entries, consuming the same underlying data
2. **Server-side aggregation API + client-side builder** (Proposal 1 of 5) — new `/api/reports/*` endpoints
3. **Gotenberg (preferred) or Puppeteer** for PDF generation (Proposal 5 of 5 for PDF)
4. **`report_templates` DB table** for saved/favorite configurations
5. **HTML/CSS templates** that double as browser preview AND PDF source
6. **All 7 PDF templates ship from day one** — full gallery, not a subset
7. **Templates are INDEPENDENT from app theme** — reports use Ternity branding, not the user's app theme
8. **In-app preview before download** — user picks template, sees preview with real data, then downloads PDF
9. **Templates tagged by medium** — "Print-friendly" vs "Screen-optimized" badges
10. **Dark templates (V2, V7 cover) labeled as screen-optimized** — V2 Dark Executive wastes ink; V1/V3/V4/V5/V6 are print-safe

## Key Reasoning: Why Templates Are Independent From App Theme

- Different audiences: app theme = personal preference, report template = client-facing brand
- Reports are formal documents — should use consistent Ternity branding regardless of user's theme
- 7 templates x 6 themes = 42 combinations to test — unsustainable
- Future "white-label" (org's own logo/colors) is a separate feature, still unrelated to app theme
