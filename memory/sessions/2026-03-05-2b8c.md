### Design Discussion — Wallchart Grouping Philosophy

User evolved their thinking about leave type groups:

1. Initially considered static groups (Wypoczynek, Zdrowie, Rodzina, etc.) by legal category
2. Then shifted to planning-horizon grouping (Planned vs Ad hoc vs Other)
3. Realized the grouping should be **derived from booking timing**, not a static property of the leave type
4. Decision: **Planned = booked 24h+ before start. Ad hoc = booked <24h before start.**
5. Partial is orthogonal — a partial booking can be planned or ad hoc
6. Leave type groups infrastructure stays in place but not populated for now
7. Wallchart should NOT show leave type names — only Planned vs Ad hoc coloring
8. Color choice: **Teal (#00D4AA)** for planned (brand, intentional), **Amber (#F59E0B)** for ad hoc (attention, unplanned)

### Session Continuation — Wall Chart Day Count Bug Investigation

User reported: Tomasz Grabowski shows "2" in the days count column on the wallchart, but only one day cell is visually marked/colored on the grid.

**Root cause:** `daysCount` stored in the DB from Timetastic sync is **wrong**. Timetastic's `deductionDays` field represents allowance deductions, not calendar working days. Examples: single-day bookings (startDate == endDate) stored with daysCount = 2, 3, 4, or 5. This is a data quality issue from the sync transform (`absences.ts` line 67-68).

**Fixes applied to `leave.tsx`:**

1. Added `countWorkingDays(startDate, endDate, holidays)` and `getBookingWorkingDays(booking, holidays)` helpers — compute actual working days from date range, excluding weekends and Polish holidays
2. Wallchart days column now uses `getBookingWorkingDays()` instead of `b.daysCount`
3. CellTooltip now computes days from dates instead of using `booking.daysCount`
4. MyLeaveList now computes days from dates instead of using `booking.daysCount`
5. Removed `booking.note` from wallchart tooltip — it was leaking leave type names (e.g., "choroba") which contradicts the Planned/Ad hoc-only design
6. All three components (WallchartGrid, CellTooltip, MyLeaveList) now receive `holidays` prop for accurate calculation

**Key insight:** Never trust `daysCount` from synced Timetastic data. Always compute from `startDate`/`endDate`.
