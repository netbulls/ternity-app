# Session 2026-03-07-9f3d

## Context

User reports that the previous session hit a "maximum of 100 PDF pages" error when trying to verify all generated PDFs. This is a Claude tool limitation — when reading PDFs, Claude can only handle up to 100 pages.

Looking at session 83a4, the QA testing was actually completed successfully using curl and file size checks (not by reading the PDFs directly). The templates generate between 9-24 pages each, so individually they're fine. But if all 7 were provided at once to Claude's PDF reader, that would be 22+22+22+23+23+9+24 = 145 pages — exceeding the 100-page limit.

## Task

User wants verification that all PDF templates generate correctly, specifically headers/footers being missing on some pages.

## Root Cause Found

When there are many team members (19 in test data), the **summary/cover pages overflow beyond A4 height**. Chromium's print engine then creates an extra physical page for the overflow content, but that overflow page has **no header or footer** because headers/footers are part of the original `.page` div structure.

### Affected Templates:

- **V4 (Magazine Spread)**: Cover page with 19-user team performance table — footer overlaps with last table rows
- **V5 (Dashboard Print)**: Page 1 with stat cards + chart + 19-user team overview card — content too tall, no footer visible
- **V6 (Invoice Style)**: Page 1 with summary + start of dense time log — content overflows to page 2

### Not Affected:

- V1 (Classic Corporate) — page 1 fits within A4
- V2 (Dark Executive) — page 1 fits within A4
- V3 (Minimal Swiss) — page 1 fits within A4
- V7 (Cover + Chapters) — cover page is simple centered content, fits fine

### Fix Strategy:

The summary/cover pages need to be smarter about how much content they include. Options:

1. Limit the team breakdown on page 1 (show top N users, "and X more..." link)
2. Split the summary across 2 pages when there are many users (**CHOSEN**)
3. Make the chart/legend section scrollable/paginated

## Fixes Applied (Session Continuation)

All three affected templates fixed with option 2 — split content to additional pages when many users present.

### V4 (Magazine Spread)

- `v4-magazine-spread.ts:604-611` — Added `V4_COVER_USER_THRESHOLD = 10`. When >10 users, the "Team Performance" section (stacked bar + summary table) is pushed to a second page as `type: 'inner'` instead of staying on the cover page. This gives it the inner-page header (logo + date) and footer with page numbering.

### V5 (Dashboard Print)

- `v5-dashboard-print.ts:476-489` — Added `V5_TEAM_USER_THRESHOLD = 10`. When >10 users, the "Team Overview" card (with avatar, name, bar, hours per user) is moved to a separate page 2 with a compact nav bar. Stat cards + chart card remain on page 1.

### V6 (Invoice Style)

- `v6-invoice-style.ts:506` — Reduced `PAGE1_ROWS` from 38 to 28. The invoice header + summary table + section title + log table header consume ~120mm, leaving only ~170mm for data rows. At ~5mm per row, 28 rows fits safely within A4 height. Continuation pages still use `CONT_ROWS = 52` (compact header takes less space).
