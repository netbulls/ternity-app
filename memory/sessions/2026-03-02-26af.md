### Timezone bug discovered

- The explore agent found a significant timezone bug: backend `GET /api/entries` uses UTC day boundaries (`T00:00:00Z` to `T23:59:59Z`) but org timezone is `Europe/Warsaw` (UTC+1/+2). Entries logged between midnight and 1-2 AM local time fall on the previous UTC day and won't appear. Same issue affects stats and dashboard routes. This explains the "missing entries" issue from prior sessions — entries created with Warsaw-midnight-adjacent times fell outside UTC boundaries.
- **Not fixed yet** — this is a real bug to address separately.

## Democaster default settings for Ternity

These are the approved defaults — use for all future Ternity demos:

**Recording:**

- Resolution: `1080p` (outputs 3840×2160 retina)
- User scale preference: `Comfortable` (1.2) in DB before recording
- Format: `mp4`

**OSD bubbles:**

```json
"osd": {
  "preset": "custom",
  "font": "Oxanium",
  "animation": "spring",
  "opacity": 1,
  "accentColor": "#00D4AA",
  "fontSize": 18,
  "customCss": "background: rgba(10, 10, 10, 0.92); backdrop-filter: blur(20px); border: 1.5px solid rgba(0, 212, 170, 0.5); border-radius: 10px; padding: 12px 18px; color: #ffffff; text-shadow: 0 1px 3px rgba(0,0,0,0.5); box-shadow: 0 4px 24px rgba(0,0,0,0.4);",
  "pointerColor": "rgba(10, 10, 10, 0.92)"
}
```

Key decisions:

- 1080p + Comfortable scale (1.2) — fonts readable, UI fills the frame nicely
- Near-opaque dark background (0.92) — text doesn't bleed through from app underneath
- Teal border accent matches Ternity brand
- Oxanium font matches app font
- Spring animation for dynamic feel
- Strong drop shadow separates bubble from app content
