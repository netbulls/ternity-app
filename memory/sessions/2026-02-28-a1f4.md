### Key discovery — Logto social identity avatars:

Logto only populates the top-level `avatar` field at account creation time. If the user created their account via phone/email and later linked Google (or if the "sync user profiles" connector setting is "only upon registration"), the top-level `avatar` stays null even though the avatar exists inside `identities.google.details.avatar`.

We decided NOT to change the Logto connector setting to "sync at each sign-in" because:

1. It would overwrite any manually-set avatar/name on every login
2. It ties behavior to a specific provider (Google) — not future-proof for Apple, Microsoft, etc.

Instead, our `/api/me` avatar refresh and JIT provisioning now check all social identities as fallback:

```
logtoUser.avatar ?? Object.values(logtoUser.identities ?? {}).find(id => id.details?.avatar)?.details?.avatar ?? null
```

This is provider-agnostic and works with any current or future social connector.
