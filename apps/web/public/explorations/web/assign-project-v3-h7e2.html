Design Notes — V3: The Split Panel

Approach: A split-panel layout with project info on the left and member management on the right. Project context (stats, description, color) is always visible while managing members. A combobox search at the bottom lets you quickly find and add users.

  • Pro: Project context always visible — admin sees member count, entry count, and project metadata while making changes.
  • Pro: Combobox pattern is familiar from tools like Linear, Notion — fast keyboard-driven workflow.
  • Pro: Inline role editing via dropdown — no modal needed. Remove buttons appear on hover to reduce clutter.
  • Pro: Drag-to-reorder hint signals future ordering capability without needing full implementation.
  • Con: Split layout uses more horizontal space — less ideal on narrow screens (collapses to stacked on mobile).
  • Con: Left panel is static info — could feel like wasted space once you've seen it.
  • Con: No bulk add/remove — still one-at-a-time operations.
