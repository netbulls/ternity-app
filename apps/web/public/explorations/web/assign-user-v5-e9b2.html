V5 — The Accordion

Projects organized by client in collapsible accordion sections. Each client section shows assignment counts and contains project cards in a 2-column grid with color bars, checkboxes, and role selectors.

Pros

  • Natural grouping by client — matches how organizations think about project ownership
  • Collapsible sections reduce visual noise — focus on one client at a time
  • Count badges on headers give instant overview without expanding anything
  • "Select All" per client enables fast bulk assignment for entire client engagements
  • Project cards offer more visual richness (color bar, spacious layout) than table rows
  • Role pill selector is more visual than a dropdown — shows both options at a glance
  • Summary bar at bottom provides running totals while making changes

Cons

  • More clicks needed — must expand each section to see/manage projects
  • Can't see all projects at once — harder to compare across clients
  • 2-column card grid uses more vertical space than a table per project
  • Sorting options are limited — projects within a section are just listed alphabetically
  • If a client has only 1-2 projects, the accordion overhead feels wasteful
  • Search/filter must work across collapsed sections (current UX shows matched sections)

Best for

  • Organizations with clear client boundaries (agency model, consulting)
  • When projects per client is 2-6 — the card grid feels right-sized
  • Admins who think in terms of "which clients should this user work on"
  • Onboarding workflows where a user is assigned to a whole client's portfolio at once
