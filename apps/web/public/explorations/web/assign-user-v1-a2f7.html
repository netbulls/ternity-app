Design Notes — V1 The Checklist

  • Immediately scannable — checked = assigned, unchecked = not. Zero learning curve.
  • Client groupings provide natural visual hierarchy for large project lists.
  • Role toggle (manager/user) sits inline — no secondary modal or step needed.
  • Search filters instantly, hiding non-matching groups. Fast for orgs with many projects.
  • Vertical scrolling grows with project count — 30+ projects would need pagination or virtual scroll.
  • Role toggle can be missed visually — only appears when a project is checked.
  • No drag-and-drop or bulk selection — assigning 10 projects means 10 clicks.
  • Dual-column layout requires wider viewport; narrows poorly on small screens.
